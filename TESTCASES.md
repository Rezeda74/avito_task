# Тест-кейсы для API микросервиса

## 1. Создать объявление
- Тест-кейс 1.1: Создание нового объявления
  - Метод: POST
  - URL: https://qa-internship.avito.com/api/1/item
  - Тело запроса:
  json
    {
        "name": "Телефон",
        "price": 15000,
        "sellerId": 345256,
        "statistics": {
            "contacts": 11,
            "like": 25,
            "viewCount":10
        }
    }
	
	В коде использованы случайно сгенерированные числа, и name из списка товаров.
	
	- Ожидаемый ответ: 201 Created
      В коде ожидаемый статус код поставлен 200, т.к. объявление уже создано	
      "status": "Сохранили объявление - afeaf0de-fa05-45f5-8640-a757b7a9ae55"
	
- Тест-кейс 1.2: Попытка создать объявление с некорректным типом данных обязательного поля
- 1.2.1 Поле name число
  - Метод: POST
  - URL: https://qa-internship.avito.com/api/1/item
  - Тело запроса:
  json
    {
        "name": случайно сгенерированное число,
        "price": случайно сгенерированное число,
        "sellerId": seller_id,
    }
  - Ожидаемый ответ: 400 Bad Request
  
- 1.2.2 Поле price строка
  - Метод: POST
  - URL: https://qa-internship.avito.com/api/1/item
  - Тело запроса:
  json
    {
        "name": случайно сгенерированное слово из списка товаров,
        "price": случайно сгенерированное слово из списка товаров,
        "sellerId": seller_id,
    }
  - Ожидаемый ответ: 400 Bad Request
  
- 1.2.3 Поле sellerID строка
  - Метод: POST
  - URL: https://qa-internship.avito.com/api/1/item
  - Тело запроса:
  json
    {
        "name": случайно сгенерированное слово из списка товаров,
        "price": случайно сгенерированное число,
        "sellerId": случайно сгенерированное слово из списка товаров,
    }
  - Ожидаемый ответ: 400 Bad Request


## 2. Получить объявление по его идентификатору
- Тест-кейс 2.1: Получение объявления по ID
  - Метод: GET
  - URL: https://qa-internship.avito.com/api/1/item/{id}
  - Параметр: {id} - существующий id объявления - afeaf0de-fa05-45f5-8640-a757b7a9ae55
  - Ожидаемый ответ: 200 OK и данные объявления

- Тест-кейс 2.2: Попытка получить объявление с несуществующим идентификатором тип строка
  - URL: https://qa-internship.avito.com/api/1/item/{случайно сгенерированная строка}
  - Метод: GET
  - Ожидаемый ответ: 400 Bad Request
  
- Тест-кейс 2.3: Попытка получить объявление с несуществующим идентификатором тип отрицательное число
  - Метод: GET
  - URL: https://qa-internship.avito.com/api/1/item/{случайно сгенерированное отрицательное число}
  - Ожидаемый ответ: 400 Bad Request
  
  
## 3. Получить все объявления по идентификатору продавца
- Тест-кейс 3.1: Получение всех объявлений по sellerId
  - Метод: GET
  - URL: https://qa-internship.avito.com/api/1/{sellerID}/item
  - Параметр: {sellerID} - существующий sellerID
  - Ожидаемый ответ: 200 OK и массив объявлений

- Тест-кейс 3.2: Попытка получить объявления по несуществующему sellerId тип строка
  - Метод: GET
  - URL: https://qa-internship.avito.com/api/1/{случайно сгенерированная строка}/item
  - Ожидаемый ответ: 400 Bad Request

- Тест-кейс 3.3: Попытка получить объявления по несуществующему sellerId тип отрицательное число
  - Метод: GET
  - URL: https://qa-internship.avito.com/api/1/{{случайно сгенерированное отрицательное число}/item
  - Ожидаемый ответ: 400 Bad Request


## 4. Получить статистику по айтем ID
- Тест-кейс 4.1: Получение статистики по айтем ID
  - Метод: GET
  - URL: https://qa-internship.avito.com/api/1/statistic/{itemID}
  - Параметр: {itemID} - существующий item ID -  afeaf0de-fa05-45f5-8640-a757b7a9ae55
  - Ожидаемый ответ: 200 OK и статистика

- Тест-кейс 4.2: Попытка получить статистику по несуществующему item ID тип строка
  - Метод: GET
  - URL: https://qa-internship.avito.com/api/1/statistic/{случайно сгенерированная строка}
  - Ожидаемый ответ: 400 Bad Request
  
- Тест-кейс 4.3: Попытка получить статистику по несуществующему item ID тип отрицательное число
  - Метод: GET
  - URL: https://qa-internship.avito.com/api/1/statistic/{случайно сгенерированное отрицательное число}
  - Ожидаемый ответ: 400 Bad Request
